Hugo 全面指南：从入门到进阶

Hugo：开启你的静态网站生成之旅 Hugo 是目前世界上最快的静态网站生成器（SSG），基于 Go 语言开发。它以极高的编译速度、强大的模板系统和简单的部署流程深受开发者喜爱。 1. 核心概念 1.1 内容 (Content) Hugo的核心是 Markdown。你只需在 content/ 目录下编写 .md 文件，Hugo 就会根据模板自动将其转换为 HTML 页面。 1.2 Front Matter 位于 Markdown 文件顶部的元数据，用于定义文章的标题、日期、标签、分类等。常见的格式有 TOML、YAML 和 JSON。 title = "我的第一篇博客" date = 2026-02-03T12:00:00+08:00 draft = false tags = ["Hugo", "Static Site"] categories = ["技术"] 1.3 模板 (Templates) Hugo 使用 Go 的 html/template 库。它允许你通过逻辑控制（如 range、if）来动态渲染内容。 2. 目录结构说明 archetypes/: 存放hugo new 时使用的内容模板。 assets/: 存储需要经过 Hugo Pipes 处理的资源（如 SCSS、JS、图片处理）。 content/: 存放所有的 Markdown 内容文件。 data/: 存放数据文件（YAML/JSON/TOML），可在模板中全局调用。 layouts/: 存放 HTML 布局模板。 static/: 存放不需要处理的静态资源（如图片、图标），构建时会直接复制到发布目录。 themes/: 存放下载或自定义的主题。 hugo.toml (或 config.toml): 整个站点的全局配置文件。 3. 常用 CLI 命令 hugo server -D: 启动本地预览服务器（包含草稿）。 hugo new posts/my-article.md: 在指定位置创建一篇新文章。 hugo: 编译整个站点，生成的静态文件会存放于 public/ 目录。 hugo list drafts: 列出所有草稿。 4. 短代码 (Shortcodes) Shortcodes 是 Hugo 提供的特殊标签，用于在 Markdown 中插入复杂的 HTML。 ...
